AutoDL GPU Container Multi-GPU Interconnect Test - Beijing Zone A - 2080Ti x2

Due to business needs, we may rent servers from AutoDL for model training, so we tested the performance of multi-GPU interconnect for consumer-grade graphics cards (original 2080Ti 11G) on their platform. The results are placed here for reference.

TL;DR: No NVLink. Although running on a PCIe 3.0x16 switch and both CPU NUMA nodes are connected to the PCIe switch, P2P communication is unavailable due to driver issues, resulting in poor multi-GPU interconnect performance.

The following is a summary from DeepSeek-R1-0528

1. Test Environment Overview

  1. Hardware Configuration:

    • GPU: NVIDIA GeForce RTX 2080 Ti × 2 (11GB VRAM each)
    • Topology: PCIe Gen3 ×16 interconnect (PIX mode shown in nvidia-smi topo)
    • NUMA Affinity: Both GPUs share the same NUMA node (Node 1)

  2. Software Environment:

    • CUDA Version: 12.4
    • Driver Version: 550.90.07
    • Testing Tools: cuda-samples, nccl-tests

2. Key Test Result Analysis

1. Basic GPU Status
  • Load Performance (nvtop):
    • Both GPUs maintain high load (GPU utilization ≈100%)
    • VRAM usage stable at 3.48GB/11GB (≈31.6%)
    • Power consumption: 91-92W (below TDP of 250W)
  • Interconnect Topology (nvidia-smi topo):
    • NVLink not supported (NS status)
    • Only connected via PCIe Gen3 ×16 (theoretical bidirectional bandwidth 32GB/s)
2. Point-to-Point (P2P) Communication Performance
  • p2pBandwidthLatencyTest:
    • P2P Access Unavailable: Device=0 CANNOT Access Peer Device=1
    • Unidirectional Bandwidth:
      • GPU0 → GPU1: 5.71 GB/s
      • GPU1 → GPU0: 5.70 GB/s
    • Bidirectional Bandwidth:
      • P2P Disabled: 6.10–6.12 GB/s
      • P2P Enabled: 6.10 GB/s (no improvement)
    • Communication Latency: 14.76 μs (GPU0→GPU1)

💡 Conclusion: PCIe interconnect bandwidth utilization is less than 20% of the theoretical value, and P2P acceleration is ineffective.

3. NCCL Collective Communication Performance
Operation TypeMax Bandwidth (busbw)Bottleneck Analysis

broadcast11.09 GB/sClose to PCIe unidirectional theoretical peak (69.4%)

all_gather7.69 GB/sData distribution efficiency lower than unidirectional operations

  • Bandwidth Patterns:
    • Unidirectional operations (broadcast/reduce) > Multidirectional operations (alltoall) > Global aggregation (all_reduce)
    • Bandwidth drops sharply for small packets (<1MB) (minimum 0.01 GB/s)
